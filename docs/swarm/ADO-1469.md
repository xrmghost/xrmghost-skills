## Goal

Configure `ossf/scorecard-action` for the `xrmghost-skills` repository, publish the score to securityscorecards.dev, and surface it via a badge in README.md. Target score >= 5.0 post-Wave-1.

### `.github/workflows/scorecard.yml`

- Triggers: `schedule` (weekly Monday 06:15 UTC), `push` to `main`, `branch_protection_rule`
- `permissions: read-all` at workflow level; job overrides with `security-events: write`, `id-token: write`, `contents: read`, `actions: read`
- Uses `ossf/scorecard-action@v2.4.0` with `publish_results: true` (public repo requirement)
- Uploads SARIF via `github/codeql-action/upload-sarif@v3` with `category: supply-chain`
- Does **not** gate / block PRs (informational only)

## Signed-Releases — N/A Justification

Il repo `xrmghost-skills` non ha release formali (è Markdown puro, nessun artefatto binario o package). Il check `Signed-Releases` di Scorecard sarà quindi N/A / non applicabile. Questo è atteso e documentato qui come giustificazione.

## Rischi Residui

- Il primo punteggio su securityscorecards.dev sarà disponibile solo dopo il merge su `main` e l'esecuzione del primo workflow (schedule o push).
- Se il repo non ha Code Scanning abilitato nelle impostazioni GitHub, l'upload SARIF potrebbe fallire silenziosamente — da verificare post-merge nelle Actions.
- `Branch-Protection` e `Webhooks` checks potrebbero abbassare il punteggio se non configurati nel repo — da gestire in sprint successivi se necessario.
