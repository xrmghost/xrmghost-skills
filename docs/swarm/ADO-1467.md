# ADO-1467

## Goal
Implementare `.github/workflows/skill-security-scan.yml` nel repo `xrmghost-skills` con upload SARIF nel Security tab e blocco dei finding critici.

## Scope e modifiche
- Confermato il file scoped `.github/workflows/skill-security-scan.yml` già presente nel branch di lavoro.
- Confermato l'uso del fallback consentito in `Risks`: `cisco-ai-defense/skill-scanner` pubblica release (`2.0.11`) ma non espone `action.yml`/`action.yaml`, quindi non risulta consumabile come GitHub Action pubblica nel formato richiesto dal prompt.
- Il workflow usa `aquasecurity/trivy-action@0.24.0` in modalità filesystem, produce `results.sarif`, carica SARIF con `github/codeql-action/upload-sarif@v3` e fallisce il job solo in presenza di finding `CRITICAL`.
- Aggiornata la branch protection di `main` per richiedere il check `scan` mantenendo i vincoli già presenti (`required_linear_history: true`, force push e deletions disabilitati).
- Aggiornata questa evidenza per il retry cycle 2 senza aprire PR e senza toccare file fuori scope.

## Rischi residui
- Il fallback con Trivy rispetta il prompt, ma non replica la semantica specifica di `cisco-ai-defense/skill-scanner` su prompt/skill content.
- Non è stato possibile verificare run GitHub Actions o alert SARIF perché il workflow non è ancora sul default branch e questo run non deve aprire PR.
- `yamllint` e `actionlint` non sono disponibili localmente; la validazione resta limitata a controllo statico del file e ai vincoli osservabili via GitHub API.
- Il test opzionale `pip install sarif-tools && pytest scripts/test_sarif_schema.py -v` non è eseguibile in questo repo: `pip` non è installato e `scripts/test_sarif_schema.py` non esiste.

## Note per review
- Il branch contiene già il workflow pronto per il merge; il retry ha aggiunto prova riproducibile e ha aggiornato la branch protection su `main` con il required check `scan`.
- Dopo il merge su `main`, il `gh run list --workflow=skill-security-scan.yml --limit 5` smetterà di restituire `404` e sarà possibile verificare il primo run e la pubblicazione SARIF nel Security tab.
- Se in futuro `cisco-ai-defense/skill-scanner` pubblicherà un vero manifest GitHub Action, il fallback Trivy potrà essere rimosso senza cambiare il contratto SARIF/branch-protection definito qui.
