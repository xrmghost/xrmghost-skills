# ADO-1471 — Commit Signing & Branch Protection

## Goal

Enable cryptographic commit and tag signing for xrmghost-skills. Set `required_signatures: true` on branch protection. Document the setup in CONTRIBUTING.md.

| File | Action |

| Criterion | Status |

## Risks & Notes

- The `required_signatures` setting is **live** from this point on. Any unsigned merge commit pushed directly to `main` will be rejected. The conductor must sign integration commits, or squash via a signed merge.
- CONTRIBUTING.md was created from scratch because ADO-1466 had not been merged into this worktree's base. If ADO-1466 is merged first, the conductor should reconcile — the content is additive (different sections).
- Build provenance attestation documented as future work (no release artefacts currently).
